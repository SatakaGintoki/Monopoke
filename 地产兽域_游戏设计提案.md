# 《地产兽域》游戏设计策划案

> **版本**：v2.0
> **更新日期**：2026-04-10
> **定位**：休闲策略 + 回合收集类游戏
> **核心玩法**：以大富翁掷骰移动为核心框架，融合宝可梦式的精灵捕捉、养成、对战系统

---

## 一、游戏概述

**游戏名称**：地产兽域（Monopoké）

**一句话简介**：和大富翁结合的宝可梦捕捉养成对战游戏

**核心特色**：
- 掷骰移动踩格子，引发各种事件
- 捕捉野生精灵，组建你的精灵队伍
- 占领地皮建造设施，靠过路费赚取金币
- 回合制战斗，用属性克制和技能打败对手

---

## 二、游戏玩法

### 2.1 开局流程

```
创建角色（输入名字，选择头像）
        ↓
选择初始精灵（火/水/草三选一）
        ↓
获得初始地皮（每人随机3块）
        ↓
获得初始金币（1000金币）
        ↓
开始游戏
```

### 2.2 回合流程

轮到你时，执行以下步骤：

```
┌─────────────────┐
│  1. 掷骰子移动   │ → 掷出点数，棋子自动走到对应格子
└────────┬────────┘
         ↓
┌────────▼────────┐
│  2. 触发格子效果  │ → 根据格子类型触发不同事件
└────────┬────────┘
         ↓
┌────────▼────────┐
│  3. 回合结束     │ → 切换到对手回合
└─────────────────┘
```

### 2.3 格子类型

| 格子 | 图标 | 效果 |
|------|------|------|
| **空地** | 🟫 | 可以购买，建造设施 |
| **野怪格** | 🔮 | 100%遇到野生精灵，可选择捕捉 or 战斗 |
| **商店格** | 🏪 | 打开商店，可买道具/精灵球/食物 |
| **事件格** | 🎰 | 随机触发好事或坏事（扣钱/得钱/精灵受伤/捕捉精灵等） |
| **营地格** | ⛺ | 选一只精灵完全回复HP，或放生精灵 |
| **地皮格** | 🏠 | 踩到对手的地皮要付过路费，踩到自己家不用 |
| **对手格** | ⚔️ | 强制与对手精灵对战 |

### 2.4 战斗流程

#### 触发方式
- 踩到 **对手地皮** 且对手有精灵驻守
- 踩到 **对手本体** 所在格子
- 踩到 **野怪格** 选择战斗

#### 战斗步骤

```
┌──────────────────────────────────┐
│        回合开始                   │
│  速度快的精灵先行动               │
└──────────────┬───────────────────┘
               ↓
┌──────────────▼───────────────────┐
│      选择行动                     │
│  ┌─────────┐  ┌─────────┐        │
│  │ 使用技能 │  │  逃跑   │        │
│  └─────────┘  └─────────┘        │
└──────────────┬───────────────────┘
               ↓
┌──────────────▼───────────────────┐
│      技能结算                     │
│  计算伤害 → 扣血 → 检查是否阵亡    │
└──────────────┬───────────────────┘
               ↓
     ┌────────┴────────┐
     ↓                 ↓
  阵亡              继续战斗
  ↓                    ↓
  精灵消失        切换到对方回合
  主人获得金币
```

#### 伤害公式
```
伤害 = (攻击方攻击 ÷ 防御方防御) × 技能威力 × 属性克制 × 随机(0.85~1.0)
```

### 2.5 地皮系统

#### 购买地皮
- 走到无主空地，弹出购买按钮
- 价格：200-500金币不等（取决于位置）

#### 升级地皮
```
空地(200金) → 小屋(500金) → 商店(1000金) → 道馆(2000金)
```

#### 过路费
- 踩到对手地皮，需要付过路费
- 费用 = 地皮等级基础费 × 驻守精灵加成
- 如果对手没有精灵驻守，费用打5折

#### 驻守精灵
- 在自己地皮上放一只精灵"驻守"
- 对手踩到这块地时，要先和驻守精灵战斗
- 战胜后不用付过路费，还能获得驻守精灵

### 2.6 经济系统

| 货币 | 用途 | 获取方式 |
|------|------|----------|
| 金币 | 购买地皮/道具/精灵球 | 每回合工资+战胜对手+卖掉道具 |
| 宝石 | 高级扭蛋 | 成就奖励（暂不开放充值） |

#### 收入来源
- **回合工资**：每走完一圈，领取 500 金币
- **战胜对手**：获得 200-500 金币
- **占领道馆**：击败道馆馆主获得 1000 金币 + 稀有精灵

### 2.7 精灵系统

#### 属性克制
```
火 > 草 > 水 > 火（循环克制）
电 > 水    冰 > 草/龙
龙 > 龙（免疫）  恶 > 幽灵/超能
```

#### 养成方式
- **升级**：战斗获得经验值，升一级全属性+5%
- **进化**：达到等级要求或使用进化道具
- **技能**：每只精灵2-4个技能，随等级解锁

### 2.8 胜负条件

| 胜利条件 | 说明 |
|----------|------|
| **精灵全灭** | 对手所有精灵都被打消失，且付不起赔偿金 |
| **金币归零** | 对手金币为0且没有任何地皮可卖 |
| **占领道馆** | 击败中心道馆的4个馆主（挑战模式） |

### 2.9 游戏模式

| 模式 | 说明 |
|------|------|
| **双人同屏** | 和朋友面对面对战 |
| **单人挑战** | 挑战AI道馆馆主 + 四大天王 |
| **排位赛** | 线上匹配其他玩家（待开发） |

## 三、游戏系统详细设计

### 3.1 系统模块总览

游戏分为 **6 大系统**，每个系统下细分多个模块：

```
地产兽域
├── 1. 地图系统
├── 2. 精灵系统
├── 3. 战斗系统
├── 4. 经济系统
├── 5. UI/表现系统
└── 6. 框架/基础设施
```

### 3.2 地图系统

| 模块 | 功能 | 复杂度 |
|------|------|--------|
| 棋盘生成器 | 随机/固定生成 4 属性区块 + 中心道馆 | 中 |
| 格子系统 | 7种格子类型（空地/精灵/商店/事件/营地/地皮/对手） | 高 |
| 地皮所有权 | 归属显示、颜色区分、驻守精灵显示 | 中 |
| 移动动画 | 掷骰→棋子移动→落点触发 | 中 |
| 地图UI | 缩放、拖拽、当前玩家高亮 | 低 |

**地图结构**：
- **中心区域**：道馆挑战区（类似终点/Boss区）
- **四大区块**：对应火/水/草/电四种属性地区，每区 6-8 块地皮

### 3.3 精灵系统

| 模块 | 功能 | 复杂度 |
|------|------|--------|
| 精灵数据表 | 50+精灵的属性/技能/进化/稀有度配置 | 高 |
| 属性克制 | 7种属性的克制关系表 | 低 |
| 数值计算 | HP/攻击/防御等公式（等级/性格/种族值） | 中 |
| 进化系统 | 等级进化 + 道具进化两种方式 | 中 |
| 技能系统 | PP值、威力、命中、属性、效果 | 中 |
| 精灵状态 | HP/SP/异常状态/增益状态 | 中 |

**精灵属性**：火、水、草、电、冰、龙、恶（7种）

**数值体系**：HP、攻击、防御、速度、特攻、特防

**稀有度**：普通 / 稀有 / 传说

### 3.4 战斗系统

| 模块 | 功能 | 复杂度 |
|------|------|--------|
| 回合管理器 | 速度排序、行动选择、回合结算 | 高 |
| 伤害计算器 | 基础伤害 × 属性克制 × 随机 × 状态修正 | 中 |
| 技能执行器 | 技能动画播放、效果触发、状态更新 | 高 |
| AI对手 | 简单AI（随机选技能）/ 策略AI（克制优先/血线判断） | 高 |
| 战斗结算 | 胜负判定、奖励发放、逃跑处理 | 低 |

**特殊机制**：
- 占领对方地皮后可派遣精灵驻守（类似收取过路费）
- 驻守精灵会随机巡逻邻近格子

### 3.5 经济系统

| 模块 | 功能 | 复杂度 |
|------|------|--------|
| 货币系统 | 金币/宝石的获取、消耗、显示 | 低 |
| 地皮购买 | 无主地皮购买逻辑 | 中 |
| 过路费计算 | 地皮等级 × 驻守精灵加成 | 中 |
| 商店系统 | 道具列表、购买逻辑、库存 | 中 |
| 扭蛋系统 | 精灵扭蛋/道具扭蛋，稀有度概率 | 中 |
| 设施升级 | 空地→小屋→商店→道馆，升级费用/效果 | 中 |

**地皮设施等级**：
```
空地 → 小屋 → 商店 → 道馆
```

### 3.6 UI/表现系统

| 模块 | 功能 | 复杂度 |
|------|------|--------|
| 主界面 | 地图视图、背包/精灵/设置入口 | 低 |
| 对战界面 | 双方精灵信息框、技能选择、战斗动画 | 高 |
| 商店界面 | 标签页切换、道具卡片、购买确认 | 中 |
| 精灵背包 | 精灵列表、详情查看、阵容调整 | 中 |
| 事件弹窗 | 机会卡/命运卡展示 + 结果 | 低 |
| 设置面板 | 音乐/音效开关、存档/读档 | 低 |
| 结算界面 | 胜负展示、排名、经验/奖励 | 低 |

### 3.7 框架/基础设施

| 模块 | 功能 | 复杂度 |
|------|------|--------|
| 存档系统 | JSON本地存档、云端同步 | 中 |
| 配置表 | Excel/CSV配置表读取（精灵数据/技能数据/道具数据） | 中 |
| 事件总线 | 模块间解耦通信 | 中 |
| 资源管理 | 精灵立绘/音效/特效的加载和缓存 | 中 |
| 多人同步 | 本地同屏（4人）/ 线上匹配（待定） | 高 |

## 第一章 游戏概述

### 1.1 一句话简介

大富翁掷骰踩格玩法与宝可梦精灵捕捉养成对战系统的融合之作，玩家在棋盘上购买地皮建造设施，同时捕捉精灵、组建队伍、与对手进行回合制战斗。

### 1.2 游戏定位

| 维度 | 描述 |
|------|------|
| **游戏类型** | 休闲策略 + 回合收集 |
| **目标平台** | PC (Windows/macOS/Linux) |
| **画面风格** | 2D Q版像素风 |
| **玩家人数** | 2人同屏对战 |
| **单局时长** | 15-20分钟 |

### 1.3 核心体验

玩家通过掷骰子在棋盘上移动，踩到不同格子触发不同事件：购买空地建造设施、捕捉野生精灵、与对手精灵对战。核心乐趣在于"资源积累"与"精灵养成"两条线的交织——既要经营地皮赚取金币，又要培养精灵队伍赢得战斗，最终达成胜利条件。

### 1.4 核心特色

1. **双系统融合**：大富翁经济系统（地皮、买卖、过路费）与宝可梦战斗系统（属性、技能、克制）的深度结合
2. **地皮驻守机制**：踩到对手地皮时，需先与驻守精灵战斗才能进入，战胜后可捕获该驻守精灵
3. **快节奏对战**：简化技能系统，每只精灵最多4个技能，降低操作门槛
4. **即时反馈循环**：每回合都有明确的收益或决策，15分钟即可完成一局

## 第二章 游戏玩法

### 2.1 开局流程

```
┌──────────────────────────────────────────────┐
│              创 建 角 色                       │
│         输入名字 → 选择头像                   │
└────────────────────┬─────────────────────────┘
                     ↓
┌──────────────────────────────────────────────┐
│            选 择 初 始 精 灵                   │
│     [小火龙 🔥]  [杰尼龟 💧]  [妙蛙种子 🌿]    │
└────────────────────┬─────────────────────────┘
                     ↓
┌──────────────────────────────────────────────┐
│            获 得 初 始 资 源                   │
│      随机3块地皮  +  1000 金币                │
└────────────────────┬─────────────────────────┘
                     ↓
┌──────────────────────────────────────────────┐
│              开 始 游 戏                       │
└──────────────────────────────────────────────┘
```

### 2.2 回合流程

每个玩家的回合按以下顺序执行：

```
┌─────────────────────────────────────────────────┐
│                   轮 到 你 了                    │
└────────────────────────┬────────────────────────┘
```

### 3.1 精灵属性

游戏包含 **7种属性**，存在以下克制关系：

```
┌──────────────────────────────────────────────────────────────┐
│                      属 性 克 制 表                           │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│    火系 🔥  > 草系 🌿  >  水系 💧  >  火系 🔥                 │
│                           ↓                                  │
│                        (循环)                                │
│                                                              │
│    电系 ⚡  >  水系 💧                                       │
│                                                              │
│    冰系 ❄️  >  草系 🌿 / 龙系 🐉                             │
│                                                              │
│    龙系 🐉  >  龙系 🐉    (免疫)                             │
│                                                              │

│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**克制倍率**：
- 正常克制：×2.0
- 被克制：×0.5
- 免疫：×0.0

### 3.2 精灵数值体系

每只精灵拥有以下 **6项基础属性**：

| 属性 | 说明 | 影响因素 |
|------|------|----------|
| HP | 生命值，降到0精灵消失 | 种族值 + 等级 |
| 攻击 (Atk) | 影响物理技能伤害 | 种族值 + 等级 |
| 防御 (Def) | 减少受到的物理伤害 | 种族值 + 等级 |
| 速度 (Spd) | 决定行动顺序 | 种族值 + 等级 |
| 特攻 (SpAtk) | 影响特殊技能伤害 | 种族值 + 等级 |
| 特防 (SpDef) | 减少受到的特殊伤害 | 种族值 + 等级 |

**数值计算公式**：
```
HP = (种族HP × 2 + 个体值 + 努力值/4) × 等级/100 + 等级 + 10
其他属性 = ((种族值 × 2 + 个体值 + 努力值/4) × 等级/100 + 5) × 性格修正
```

**性格修正**：
- 修正 ×1.1：孤独、勇敢、固执、开朗、天真、爽朗、慎重、浮躁
- 修正 ×0.9：慢吞吞、胆怯、开朗、温柔、温顺、忧郁、急躁、马虎

### 3.3 精灵稀有度

| 稀有度 | 比例 | 特征 | 示例精灵 |
|--------|------|------|----------|
| **普通** | 60% | 基础数值 | 绿毛虫、独角虫、波波 |
| **稀有** | 30% | 较高数值 + 外观 | 小火龙、杰尼龟、妙蛙种子 |
| **传说** | 10% | 最高数值 + 专属技能 | 小火龙(闪光)、迷你龙、利欧路 |

### 3.4 精灵养成

#### 3.4.1 升级

### 4.3 伤害公式

```
伤害 = floor( floor( (攻击方攻击 × 技能威力) / 防御方防御 ) × 属性克制 × 随机系数 )

其中：
  - 随机系数 = random(0.85, 1.0)
  - 属性克制 = 2.0 (克制) / 0.5 (被克制) / 0.0 (免疫)
  - floor() 表示向下取整
  - 最终伤害最低为 1
```
